# 🎙 Транскрибатор

Локальная система распознавания речи с диаризацией спикеров. Написано в 5 утра студентом, который очень хотел спать, но нужно было сдать проект.

## 🤔 Что это вообще такое?

Это веб-приложение, которое берет аудиофайл и выдает:
- Текст того, что сказали
- Кто что сказал (разделение по спикерам) 
- Временные метки
- **Автоматическое структурированное саммари** с разбивкой по темам, решениям, задачам и рискам (если подключена LLM)

Работает полностью локально, никуда ничего не отправляет.

### ✨ Новое: Автоматическое саммари

После завершения транскрибации система **автоматически** генерирует структурированное саммари через LLM (Qwen2.5-7B):

- 📋 **Краткое резюме** - суть разговора в 2-3 предложениях
- 🎯 **Темы** - ключевые топики с категориями (технические, бизнес, организационные)
- ✅ **Решения** - что было решено, кто ответственный
- 📝 **Задачи** - action items с исполнителями и сроками
- ⚠️ **Риски** - выявленные проблемы и ограничения
- 💡 **Ключевые моменты** - важные факты, которые нельзя упустить

Все в JSON формате, готово для интеграции в любую систему.

## 🏗 Архитектура (почему именно так)

### Основные компоненты

1. **FastAPI** - веб-сервер
2. **Whisper** - распознавание речи (2 варианта)
3. **Resemblyzer** - определение спикеров по голосу
4. **Qwen2.5-7B** - суммаризация разговоров (опционально)

### Почему именно Whisper?

Изначально хотел использовать обычный OpenAI Whisper, но он плохо работал с русской речью низкого качества (телефонные звонки). Поэтому выбрал:

- **Основной**: `bond005/whisper-podlodka-turbo` - дообученная модель на зашумленной русской речи
- **Бэкап**: `faster-whisper` с моделью `large-v3` - если HuggingFace модель не загрузится

### Диаризация спикеров

Тут два подхода:

#### 1. Стерео режим (идеально для телефонии)
Если файл стерео и в каждом канале есть звук - считаем что каждый канал = один спикер. Транскрибируем каждый канал отдельно, потом склеиваем по времени.

**Плюсы**: 100% точность разделения
**Минусы**: работает только если запись сделана правильно

#### 2. Моно режим (универсальный)
Используем Resemblyzer для определения спикеров по эмбеддингам голоса:

1. Транскрибируем весь файл Whisper'ом
2. Для каждого сегмента извлекаем эмбеддинг голоса
3. Кластеризуем эмбеддинги (жадный алгоритм с центроидами)
4. Сглаживаем результат (мажоритарное голосование)
5. Убираем мелкие кластеры

**Настройки диаризации** (можно крутить):
- `DIAR_THRESHOLD = 0.55` - порог для создания нового спикера (больше = больше спикеров)
- `DIAR_MIN_SEG_SEC = 0.8` - короткие сегменты расширяем контекстом
- `DIAR_CONTEXT_SEC = 2.0` - максимальный контекст для эмбеддинга
- `DIAR_SMOOTH_WINDOW = 3` - окно сглаживания

### Телефонная версия (app_telephony.py)

Отдельная версия для телефонного трафика 8kHz:

1. **Предобработка аудио**:
   - Bandpass фильтр 300-3400 Hz (телефонный диапазон)
   - Noise gate для удаления фона
   - Качественный апсемплинг 8kHz → 16kHz

2. **Оптимизация для M3 Mac**:
   - MPS для предобработки (PyTorch)
   - CPU для инференса (ctranslate2 не поддерживает MPS)
   - int8 квантизация для скорости
   - Использование всех ядер процессора

### LLM суммаризация

Использую Qwen2.5-7B в формате GGUF через llama-cpp-python:

- Модель: `bartowski/Qwen2.5-7B-Instruct-GGUF`
- Квантизация: Q4_K_M (баланс качества и скорости)
- Контекст: 8192 токена
- Промпт: заточен под деловые переговоры и техническое обсуждение

**Почему Qwen**: хорошо понимает русский, быстро работает, умеет в JSON.

## 🚀 Развертывание

### Docker (рекомендуется)

```bash
# Клонируем
git clone <repo>
cd transcriber

# Запускаем
docker-compose up --build

# Открываем http://localhost:8000
```

Docker автоматически:
- Скачает все модели при первом запуске
- Настроит CUDA если есть GPU
- Создаст volume для моделей

### Ручная установка

```bash
# Python 3.10+
pip install -r requirements.txt

# Для CUDA (опционально)
pip install torch torchaudio --index-url https://download.pytorch.org/whl/cu121

# Для LLM (опционально)
CMAKE_ARGS="-DLLAMA_CUBLAS=on" pip install llama-cpp-python

# Скачиваем модели
huggingface-cli download Qwen/Qwen2.5-7B-Instruct-GGUF qwen2.5-7b-instruct-q4_k_m.gguf --local-dir models/qwen/

# Запускаем
python app.py
```

### Переменные окружения

```bash
# Whisper
WHISPER_HF_MODEL="bond005/whisper-podlodka-turbo"  # HF модель
WHISPER_MODEL="large-v3"                           # бэкап модель
USE_HF_MODEL="1"                                   # 0 = только faster-whisper

# LLM
LLM_MODEL_PATH="/models/qwen/Qwen2.5-7B-Instruct-Q4_K_M.gguf"
LLM_N_GPU_LAYERS="-1"                              # -1 = все слои на GPU
LLM_N_CTX="8192"                                   # размер контекста

# Сервер
PORT="8000"
```

## 📱 Использование

### Веб-интерфейс

1. Открываем http://localhost:8000
2. Перетаскиваем аудиофайл или выбираем через кнопку
3. Жмем "Распознать"
4. Ждем (время зависит от длины файла и железа)
5. Получаем результат с разбивкой по спикерам + **автоматическое саммари** (если LLM загружена)

### API

### Структура саммари

Саммари автоматически генерируется после транскрибации и включает:

- **summary** - краткое резюме всего разговора (2-3 предложения)
- **topics** - ключевые темы с категориями (technical/business/organizational/decision/problem)
- **decisions** - принятые решения с контекстом и ответственными
- **action_items** - конкретные задачи с исполнителями и сроками
- **risks** - выявленные риски и проблемы
- **key_points** - самые важные факты из разговора

### Поддерживаемые форматы

- MP3, WAV, M4A, OGG, FLAC, WEBM
- Моно/стерео
- Любой sample rate (автоматически ресемплируется в 16kHz)

## 🔧 Настройка под свои нужды

### Точность диаризации

Если спикеров определяется неправильно:

```python
# В app.py
DIAR_THRESHOLD = 0.45      # меньше = меньше спикеров
DIAR_SMOOTH_WINDOW = 5     # больше = более плавное переключение
```

### Качество транскрибации

```python
# В _transcribe_channel()
temperature=[0.0]          # детерминированный результат
beam_size=1               # быстрее, но хуже качество
no_speech_threshold=0.8   # больше = меньше ложных срабатываний
```

### Производительность

```python
# GPU память
LLM_N_GPU_LAYERS = 20     # только часть слоев на GPU

# CPU
LLM_N_THREADS = 4         # меньше потоков для CPU
```

## 🐛 Траблшутинг

### "CUDA out of memory"
```bash
# Уменьшаем слои на GPU
export LLM_N_GPU_LAYERS=20

# Или отключаем GPU для LLM
export LLM_N_GPU_LAYERS=0
```

### "Model not found"
```bash
# Проверяем путь
ls -la /models/qwen/

# Скачиваем заново
huggingface-cli download Qwen/Qwen2.5-7B-Instruct-GGUF qwen2.5-7b-instruct-q4_k_m.gguf --local-dir models/qwen/
```

### Плохое качество для телефонии
```bash
# Используем телефонную версию
python app_telephony.py
```

### Медленно работает
```bash
# Отключаем HF модель
export USE_HF_MODEL=0

# Используем меньшую модель
export WHISPER_MODEL=medium
```
